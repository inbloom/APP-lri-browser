about inBloom Browser

inBloom BrowserTM is a tool to find educational resources on the web that is more effective and targeted than using a search engine.  This user guide describes how to use the Browser web application.

This guide applies to the 1.2 release of inBloom Browser. This version of the inBloom Browser works best in Google Chrome and Firefox on Mac and Windows, but it should work well in all modern, HTML5 compliant browsers.

The user should be on a public internet facing network or, to use the web application from behind a firewall proxy, the user should have a proxy configured to allow traffic to the inBloom Browser URL: http://browser.inbloom.org.

Version 1.2 Release Notes

May 6, 2013

INTRODUCTION

Browser V1.2 is a new way to explore educational resources.  With it, the user can both browse and search for educational resources that are tagged with Learning Registry Metadata Index (LRMI) data, many of which are aligned with the Common Core State Standards (CCSS).

Browser is an application released by http://www.inbloom.org that works with the inBloom technology stack known as the Learning Registry Index (LRI).

The inBloom Browser is found at http://browser.inbloom.org.

More information about LRMI at http://www.lrmi.net as well as http://www.schema.org/CreativeWork. As of April 2013 most of the LRMI V1.1 specification was officially adopte by schema.org.

When the user first arrives at Browser, the user can browse for all educational resources that are aligned with either CCSS for Math or for English and Language Arts (ELA).  Do so the user clicks on either the Mathematics or English and Language Arts  button at the top left of the screen. (Browser opens with Mathematics selected as the default.) Then the user  browses through the standards on the left of the screen. For each standard, Browser will provide a list of education resources aligned with that standard.

FILTERING BROWSE RESULTS

The user can filter the browse results in a number of ways:

  1. Move the K-12 slider in the top-center of the screen to restrict available standards to the grade(s) of your choice.
  2. Click on any of the primary filters:
  1. For Teachers
  1. Shows resources specifically tagged as intended for teachers.
  1. For Students
  1. Shows resources specifically tagged as intended for students.
  1. Reading
  1. Shows resources specifically tagged as reading.
  1. Audio/Video/Online
  1. Shows resources specifically tagged as audio, or video, or online

SEARCH

The user can also search for items using free text search by typing text in the field in the upper right corner of the screen.  

The user can filter the free text search results using the filter facets. To view the filter facets, the user clicks on the half-moon “filter” blue button on the left, which opens an accordian style group of filters that narrow the search when selected.  Items within groups are “or’d” together.  Groups are “and’ed” together.

For example, Clicking on For Students as well as Reading filters results to show those items that are marked as For Students AND as Reading, but it will not show results marked as audio/video/online.  

Item’s chosen within a facet are OR’ed.  For example, if the user chooses Students and Teachers within the Intended End User Role facet while searching, Browser will provide results for Students or Teachers. If the user then chooses Reading, Browser will return results that marked as Reading AND (Students OR Teachers).

The filter facet groups are defined by LRMI and items within the groups are primarily defined LRMI vocabularies.

VIEWING RESOURCES

To view a resource, the user can click on a resource tile to be redirected to that resource’s web page.

The user can also view information about the resource by clicking on the tile’s “i” button which will provide a description of the resource, the learning standards alignments to that resource, LRMI tags for that resource, etc.  The user can also bookmark the resource, send the resource to a friend, and like the resource.

WHERE DOES THE INFORMATION FOR RESOURCES COME FROM?

The information for resources seen in the Browser comes from the Learning Registry (LR), http://www.learningregistry.org/. The LR is a repository of metadata information on educational resources to which anyone can contribute.

Only resources in the LR that are marked as LRMI get filtered in inBloom’s LRI.  The Browser communicates with the LRI to navigate to the resources requested by the user. Resources that are aligned with one or more CCSS standards are shown when browsing either Mathematics or Language Arts.  All LRMI resources are shown when performing a free text search whether or not they are assigned to a CCSS standard.

INFORMATION TRACKED

When the Browser is used, the text that the user searches for (and only the text) is sent to the Learning Registry. The Browser uses the LR to store this type of data called paradata. It is anticipated that in the future, programs will use this paradata to provide insight into what kinds of resources are being searched so that better suggestions can be made.  

Also, when liking a resource, a message is stored in the Learning Registry that a user liked this particular resource. It is anticipated that in the future, developers will create programs to use that information.

No other information is sent to the LR. No personally identifiable information is ever sent.

For all data sent to the LR, the LR is only used as a repository; the LR does not act upon the data.

The Browser also uses Google Analytics for standard web analytics.

Browser FAQ

Q: What is the Browser?

A: The Browser is a web application that allows users to search for educational resources that are tagged with the Learning Registry Metadata Index (LRMI) meta data schema. Standards may or may not be aligned with the Common Core State Standards (CCSS).

Q: What problem is Browser solving?

A: Browser and the software infrastructure upon which it rests have been developed to make it easier to find educational resources on the web than existing search engine technologies.

Q: What is the Learning Resource Metadata Initiative (LRMI)?

A: LRMI is a project co-led by the Association of Educational Publishers and Creative Commons to create a standard metadata framework for tagging educational resources on the web for search, discovery, and delivery of educational content that is easier to find than current search engine technologies. For details, see http://www.lrmi.net.  Most of LRMI has been adopted by schema.org, an effort by search engines to develop schemata that can make web pages more accurate so that search results are more targeted.

Q: What is the Learning Registry (LR)?

A: The LR is a repository for information on educational resources. It’s a joint venture between the Department of Defense and the Department of Education. See:  http://www.learningregistry.org/ for more information.

Q: What are the Common Core State Standards (CCSS)?

A: The Common Core State Standards (CCSS) are an enumerated list of learning objectives that, taken as a whole, are minimum proficiency standards needed by a graduating US high-school senior to be ready for university study. The CCSS was developed by the Council of Chief State School Officers and has been adopted by a majority of states.  See http://corestandards.org for more information.

Q: What is the Learning Registry Index (LRI)?

A: The LRI is infrastructure software produced by inBloom that is used by application developers to create a variety of programs from information contained in the Learning Registry. Currently, it provides access to information about learning resources that are tagged with LRMI meta data. The resources may or may not be aligned with the Common Core State Standards (CCSS). The LRI can provide text searches on all LRMI tagged learning resources that are in the Learning Registry.  If a resource is also aligned with a CCSS standard, the LRI can also provide relationships via a graph database.

Q: How can users tag information on learning resources so that they appear in the Browser?

A: Anyone can tag educational resources by using Tagger, an inBloom program that can be found at http://tagger.inbloom.org. To publish metadata tags on learning resources to the LR, the user will need to register for a free inBloom account by visiting https://www.inbloom.org/for-developers.

Q: I’m a developer. How can developers find out more about the LRI?

A: You can find out about the LRI and inBloom’s open source software initiatives at the inBloom developer website at https://www.inbloom.org/for-developers.

Q: Does the Browser track any of my information as I use it?
A: The Browser does track information, but only anonymously.  The Browser does store paradata in the LR.  For example, the Browser does store all free text searches into the LR so that developers in the future can use that paradata to provide insight into the most popular items being searched.  The Browser also uses Google Analytics to provide usage insight to inBloom.

Q: Why are only Mathematics and Language Arts selectable filters for learning standards?

A: The LRI Browser and the LRI infrastructure currently only “knows” about the CCSS Standards. CCSS standards only currently exist for Mathematics and Language Arts.

Q: Why did you choose the specific filters that appear on the “filters” button when performing a text search?

A: The filter categories are defined the LRMI.  The choices within those categories are LRMI defined vocabularies for each of those categories.

Q: Why don’t I see any results?

A: If you do not see any results, then there are no LRMI resources in the LR that are tagged with the keywords you were searching for or aligned to the CCSS standard that you are viewing. These empty searches are opportunities for educational publishers, teachers, curriculum specialists, etc., to tag information for those particular searches, thereby assuring that others will be able to view their favorite resources.
